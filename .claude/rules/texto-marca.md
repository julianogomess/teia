# Regras — texto e marca

- Texto voltado a usuário ou organização cliente (UI, system prompt, README, docs): usar a skill `tom-teia` antes de escrever; antes de commitar, rodar o agente `revisor-de-marca`.
- Vocabulário-âncora (context/brand.md §3.2): propósito, impacto, missão, capacidades humanas, supervisão. Nunca vender IA como solução mágica nem substituta de julgamento humano; sem hype ou urgência artificial.
- Falar como quem vem do setor de impacto, não como fornecedor de tecnologia genérico.
- Identidade visual (paleta, tipografia) definida em context/brand.md — não inventar estilos novos.
- Doc nova: enxuta, sem duplicar conteúdo existente — linkar a fonte. Datas sempre absolutas.
